# Интеграции с почтой и календарём

Каждый зарегистрированный сотрудник может открыть `/integrations` и сохранить
свой провайдер и идентификатор аккаунта.

Поддерживаемые направления заготовки:

- почта: Google Gmail, Microsoft 365, SMTP;
- календарь: Google Calendar, Microsoft 365, CalDAV.

Текущий статус `pending` означает, что метаданные сохранены, но авторизация у
провайдера ещё не выполнена. Пароли, access token и refresh token в SQLite не
хранятся.

Для рабочего подключения следующему этапу потребуются OAuth-приложения Google
и/или Microsoft либо серверные настройки SMTP/CalDAV. Секреты следует хранить
в системном хранилище секретов, а в базе — только непрозрачную ссылку на них.

Контракты будущих провайдеров находятся в `core/integrations/ports.py`.
Персональные настройки и бизнес-логика находятся в
`core/integrations/service.py`, Telegram-интерфейс — в
`core/integrations/telegram.py`.
