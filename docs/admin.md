# Руководство администратора

## Настройка и запуск

Первичная подготовка изолированной среды:

    powershell -ExecutionPolicy Bypass -File .\onBots.ps1

Запуск:

    powershell -ExecutionPolicy Bypass -File .\onBots.ps1 -RunBot

Основные настройки `.env`:

    TELEGRAM_BOT_TOKEN=telegram-token
    DATABASE_PATH=data/base.sqlite3
    APP_TIMEZONE=Europe/Moscow
    ADMIN_TELEGRAM_ID=123456789
    ADMIN_FULL_NAME=Иванов Иван Иванович
    DEFAULT_REMINDER_DAYS=14
    DEFAULT_REMINDER_TIME=09:00
    DEFAULT_REMINDER_TEXT=Напоминание: ваш отпуск начинается {start_date}.

Администратор создаётся автоматически при инициализации новой БД. `.env` не добавляется в Git.

## Сотрудники и отпуска

- `/employees` — список сотрудников. После выбора доступны просмотр и добавление отпусков;
- `/export 2026` — XLSX-выгрузка, открываемая Microsoft Excel;
- `/reminder ...` — настройка отпускных напоминаний.

Отдельной команды `/employee` нет: просмотр выполняется через `/employees`. `/export`, `/broadcast` и `/reminder` доступны только администратору.

Администратор получает предупреждения, если:

- суммарный отпуск сотрудника превышает 28 календарных дней за год;
- один период превышает 28 дней;
- отпуск оформлен на один день;
- начало или окончание приходится на выходной;
- отпуск разбит на три и более периодов;
- отпуск внесён менее чем за 14 дней;
- между периодами не более двух дней.

Предупреждение не блокирует сохранение отпуска.

## Планируемые общие уведомления

Уведомления хранятся в таблице `scheduled_notifications` базы SQLite. Можно создать любое количество записей. Фоновый планировщик каждую минуту отправляет наступившие уведомления всем активным зарегистрированным пользователям и сохраняет количество доставок и ошибок.

Создать уведомление:

    /broadcast 25.12.2026 09:30 Сегодня сокращённый рабочий день

Посмотреть очередь:

    /broadcast list

Отменить ещё не отправленное уведомление:

    /broadcast delete 3

После отправки запись остаётся в БД со статусом `sent`. Отменённая запись получает статус `cancelled`. Переменная `BROADCAST_MESSAGE` больше не используется.
