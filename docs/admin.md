# Руководство владельца продукта

## Команды владельца

| Команда | Назначение |
|---|---|
| /staff | Полный список сотрудников, редактирование ролей и профилей, удаление записей |
| /guest | Пошаговое создание гостя с выбором руководителя |
| /notifications | Создание, просмотр, настройка повторов и удаление нотификаций |
| /team_create | Создание команды |
| /invite_team | Добавление сотрудника в выбранную команду |
| /dismiss_team | Исключение сотрудника |
| /delete_team | Удаление команды |
| /export | Выгрузка отпусков за выбранный год или за всё время |

Команды /broadcast и /reminder удалены. Их функции объединены в /notifications.

## Нотификации

При создании выбираются аудитория, конкретная команда либо один или несколько сотрудников, дата первой отправки, текст, периодичность и количество отправок. Доступны разовая отправка, каждый час, каждый день и каждую неделю.

Список запланированных нотификаций позволяет изменить периодичность и число оставшихся отправок либо удалить запись.

Руководитель может создать разовое оповещение только для своей команды из /employees.

## Гости

Создание гостя проходит последовательно: руководитель, ФИО, телефон, локация, грейд, направление и рабочий Email.

## Удаление сотрудника

Команда /staff позволяет открыть полное управление записью. Удаление требует подтверждения и удаляет связанные отпуска и членство в командах.
## Логирование

Успешный входящий апдейт создаёт одну компактную запись без текста сообщений. Ошибки Telegram и обработчиков журналируются отдельно со stack trace. Техническая телеметрия записывается раз в пять минут и содержит память процесса, CPU, свободное место и размер БД.
