# Руководство владельца

## Настройка и запуск

```powershell
powershell -ExecutionPolicy Bypass -File .\onBots.ps1
powershell -ExecutionPolicy Bypass -File .\onBots.ps1 -RunBot
```

Основные параметры находятся в `.env`. Полный список фич-тоглов приведён в `.env.example`. После изменения ролей или свойства тимлида перезапустите бота, чтобы Telegram заново назначил персональные меню команд.

## Ролевая модель

- **Владелец** управляет всеми сотрудниками, ролями, командами, отпусками и рассылками.
- **Тимлид** — свойство сотрудника. Тимлид управляет закреплённой командой и её календарями.
- **Сотрудник** управляет профилем и своим календарём.
- **Гость** использует свой календарь и видит контакт тимлида.

Команды `/broadcast`, `/export` и `/guest` регистрируются в меню Telegram только для владельца.

## Команды тимлидов

| Команда | Формат |
|---|---|
| `/team` | Карточка команды и список участников |
| `/team_create` | `/team_create Название команды` |
| `/team_add` | `/team_add ID_команды ID_сотрудника` |
| `/team_remove` | `/team_remove ID_команды ID_сотрудника` |
| `/employees` | Управление закреплёнными сотрудниками |

Владелец создаёт команду для конкретного тимлида командой `/team_create ID_тимлида Название команды`. Один сотрудник состоит только в одной команде. При переносе в другую команду связь с тимлидом обновляется автоматически.

## Команды владельца

| Команда | Назначение |
|---|---|
| `/guest [ID_тимлида] Фамилия Имя` | Создать гостя |
| `/export [год]` | Выгрузить отпуска в XLSX |
| `/broadcast ...` | Создать или изменить рассылку |
| `/reminder ...` | Настроить напоминания |

Рассылка:

```text
/broadcast [roles=employee,team_lead] ДД.ММ.ГГГГ ЧЧ:ММ текст
/broadcast list
/broadcast roles ID employee,guest
/broadcast delete ID
```

```powershell
python start.py database backup
.\.venv\Scripts\python.exe -m pytest -q
.\.venv\Scripts\python.exe -m ruff check bot
```

Перед обновлением сделайте резервную копию файла SQLite из `DATABASE_PATH`.
